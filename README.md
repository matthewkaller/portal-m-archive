# portal-m-archive

本仓库用于归档和发布多个独立的 HTML 页面。这些页面可以是静态展示、工具界面、信息汇总或其他形式的 Web 内容，不针对任何特定域名或网站。

## 项目简介

portal-m-archive 是一个通用型 HTML 页面归档仓库。其目的是将分散的独立页面集中存储、版本管理和公开访问，方便维护与分享。仓库本身不提供搜索、索引或内容推荐功能，仅作为静态资源的载体。

## 目录结构

```
portal-m-archive/
├── pages/          # 存放所有归档的 HTML 页面文件
├── assets/         # 页面所需的公共资源（CSS、JS、图片等）
├── index.html      # 可选：仓库根目录的入口页面
└── README.md       # 本文件
```

- **pages/**：每个子目录或文件代表一个独立的归档页面，命名遵循内容主题或日期。
- **assets/**：存放页面间共享的样式、脚本或媒体文件，避免重复上传。
- **index.html**：如存在，可用于展示仓库内容概览或导航。

## 页面归档说明

- 所有页面均为独立的 HTML 文件，可离线或在线直接打开。
- 页面间无强制依赖关系，每个页面可独立运行。
- 归档内容可能随时间更新，历史版本可通过 Git 提交记录查看。
- 文件命名建议使用英文小写字母、数字和连字符，避免特殊字符。

## 维护说明

- 欢迎提交 Pull Request 添加新的独立页面，或对现有页面进行改进。
- 新增页面请放置在 `pages/` 目录下，并确保路径清晰。
- 请勿在页面中包含恶意代码、跟踪脚本或违反法律法规的内容。
- 如有公共资源需求，请优先放入 `assets/` 目录，并在页面中引用相对路径。

## 使用方式

1. 克隆本仓库到本地：
   ```
   git clone https://github.com/your-username/portal-m-archive.git
   ```
2. 直接在浏览器中打开 `pages/` 下的 HTML 文件，或通过本地服务器预览。
3. 如需部署到静态托管服务（如 GitHub Pages），请将仓库设置为 Pages 源，访问根目录或 `pages/` 路径。
